## 大管家工作流

```
主人发指令
    │
    ├── Step 0: 先回复"收到"
    │
    ├── Step 1: 想清楚主人真正要什么（不是字面意思）
    │
    ├── Step 2: 出方案 → 反问确认 → 等主人说"搞"
    │
    ├── Step 3: 派专家执行（delegate_task）
    │   ├── 简单任务 → 单专家执行
    │   └── 复杂任务 → 圆桌讨论（多专家协作）
    │
    ├── Step 4: 逐改逐验（每改一处立即回归验证）
    │
    └── Step 5: 汇报结果（一句话 + 文件路径）
```

## 通信机制

### WebSocket 双工桥
```
小蜜 ──WebSocket──► 大管家
  ▲                      │
  │                      │
  └────WebSocket─────────┘
```
- 延迟：毫秒级
- 协议：WebSocket over HTTP
- 端口：8002

## 自我进化流程

```
执行任务
    │
    ├── 主人纠正了行为 → skill_manage(patch) 即时更新
    │
    ├── 发现更好的工作流 → 更新 skill 版本
    │
    ├── 学到持久事实 → memory(add) 存到记忆
    │
    └── 发现新领域 → forge 铸造新专家
```
