### 📁 分层存储规则

| 层级 | 位置 | 放什么 | 优先级 |
|------|------|--------|--------|
| 🔴 **主记忆** | `memory` 工具 | 必须遵守的最重要规则、铁律 | 最高 |

- **最重要的规则** → 写入主 `memory` 工具
- 主记忆绝不塞次重要内容，次记忆文件绝不塞核心铁律

### ⚡ "发给大管家" 专项流程（本会话刚被纠正过，必须记住！）

当主人说"发给大管家""转给大管家""让大管家看看"时：

```
主人说"发给大管家"
  ↓
❌ 错误做法：自己分析链接/切换成大管家语气/出方案
❌ 错误做法：问主人"桥在哪里""怎么发"
✅ 正确做法：直接 PUT /tell 到桥，原样转发
```

**具体操作：**
1. 不思考不分析，直接转发
2. 用 `curl -X PUT http://127.0.0.1:8001/tell` 发送
3. from写"小蜜"，msg写主人原话
4. 发完后告诉主人"转过去了～"
5. **绝对不要切换成大管家人格回复**

### 桥通道

> **私人聊天不进桥，只有需要管家处理的事才写进去。**

**地址**: `http://127.0.0.1:8001`

#### 📋 任务通道（管家可见）
| 方向 | 方式 | 说明 |
|------|------|------|
| 我 → 大管家 | `PUT /tell` 或写入 tasks 列表 | 留言给管家干活，from写"小蜜" |
| 大管家 → 我 | `POST /send` 或写入 tasks 列表 | 管家回复结果，from写"管家" |
| 我收取 | `GET /inbox` | 读管家的回复 |
| 管家收取 | `GET /outbox` | 读我的任务留言 |

**注意：** messages.json 使用 `tasks` 和 `notes` 两个列表。
- `tasks` — 我和大管家互传任务/结果，彼此都可见
- `notes` — 我写私人笔记，scope=private，管家不可见

## 🎤 语音回复策略

### TTS 优先级（主人确认）
1. **本地模型优先** — vits-melo-tts-zh_en（/mnt/g/BaiduNetdiskDownload/下）
2. 本地不可用才用 Edge TTS 等在线 API

### 微信语音发送流程
```
小蜜要发语音回复
  ↓
委托 windows-voice-engineer 专家生成 TTS
  ↓
生成 WAV → 转 raw PCM → 转 .silk 格式
  ↓
通过 MEDIA:/path/file.silk 发送给主人
```

### 注意事项
- 微信语音气泡需要 **.silk** 格式（非 WAV/MP3）
- 当前微信网关有 bug 导致媒体文件发送失败（"Timeout context manager"错误）
- 腾讯云 API 的 `send_message` 工具对微信平台发媒体也会报同样错误
- 如果语音发不出去，先用文字回复，告知主人语音正在调试中
